### C-010 — Dashboard Nav Restructure (new, completed this session)

All implemented in a new standalone file `app/public/dashboard-nav.js` injected via `app.blade.php` and `Dockerfile`.

#### Team tab moved to header nav

- `.dashboard__routes` (the built-in Timeline/Team tab bar) hidden globally via CSS
- A `<li id="dn-team-nav-item">` is injected into `.at-menu.navbar > div` after the Dashboard item
- Visibility gated on `user.can_view_team_tab` (true for Admin/Manager/Auditor, false for Employee)
- Click wired to `vm.$router.push({ name: 'dashboard.team' })`
- `localStorage.dashboard.tab` is locked to `'timeline'` whenever not on the Team page, defeating the compiled router's `beforeEnter` guard that would otherwise redirect `/dashboard` to Team

#### Active state management

- `body.dn-on-team` toggled when on `/dashboard/team` or `/team`
- CSS suppresses Dashboard's `router-link-active` and `at-menu__item--active` highlight when on Team page — both the `<li>` and `<a>` classes are handled since AT-UI applies both
- Team link gets `.dn-active` class with blue underline indicator

#### Dashboard → Timeline click fix

- AT-UI suppresses clicks on already-active parent nav items
- Fixed with a capture-phase listener (`stopImmediatePropagation + preventDefault`) that pushes `{ name: 'dashboard.timeline' }` directly, bypassing AT-UI's handler

#### Projects dropdown flattened

- The Projects submenu (Projects + Project Groups) is detected by scanning `.at-menu__submenu` titles for "project"
- Original dropdown is hidden; a plain `<li id="dn-projects-link">` direct link to `/projects` is injected before it
- Projects active state (`body.dn-on-projects`, `.dn-active`) managed in `updateActiveState()`

#### Group column removed from Projects page

CSS grid template custom properties overridden with `!important` to account for the missing column (4→3 data columns), and `body.dn-on-projects .at-table tr > *:nth-child(2) { display: none !important; }` hides the Group cell in every row including the header.

#### Tasks list limited to 5 rows

- `body.dn-on-tasks .at-table__body tr:nth-child(n+6) { display: none !important; }` — hides rows 6+
- `body.dn-on-tasks .at-pagination { display: none !important; }` — hides pagination
- `injectTasksHint()` injects a `<p>` after the table: *"Showing 5 most recent tasks. Use the search above to find others."*

#### Add Time + Import buttons hidden on dashboard

- `body.dn-on-timeline` class added when on `/dashboard`, `/dashboard/timeline`, `/timeline`, or `/`
- CSS: `body.dn-on-timeline .controls-row { display: none !important; }` — removes both buttons
- Buttons remain visible on Team page and other pages where they're relevant

#### Tasks page — user avatars capped at 2 + "+N" badge

- Root cause: Projects page uses the native `TeamAvatars` Vue component (limits to 2 natively); Tasks page renders all users in an uncapped `div.flex.flex-gap.flex-wrap`
- `limitTaskAvatars()` finds `.flex.flex-gap.flex-wrap` containers on `/tasks`, hides direct children past index 2, appends a `div.dn-extra-badge` styled to match `team-avatars__placeholder` (30×30px, `#9e9e9e` bg, `border-radius: 5px`)

## Key Technical Notes

- **Scoped CSS can't be reused**: `team-avatars__placeholder` CSS uses Vue scoped attribute selectors (`[data-v-34a45e46]`) — injected DOM nodes don't carry that attribute, so styles must be inlined
- **AT-UI active state uses two classes**: `at-menu__item--active` on `<li>` AND `router-link-active` on `<a>` — both must be suppressed when overriding active state
- **Capture-phase listeners beat AT-UI**: AT-UI registers click handlers in bubble phase; adding `{ capture: true }` + `stopImmediatePropagation` lets our handler win
- **`localStorage.dashboard.tab` guard**: the compiled router's `beforeEnter` on `/dashboard` reads this key and redirects to `/dashboard/team` if the value is `'team'` — locking it to `'timeline'` on every tick defeats this without touching compiled code
- **Tasks Users column structure** (from compiled `app.js`): `div.flex.flex-gap.flex-wrap` > `AtTooltip` (one per user) > `UserAvatar` (`vue-avatar--wrapper`) — no built-in truncation
