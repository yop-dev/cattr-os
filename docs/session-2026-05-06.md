## Key Technical Notes (additions)

### `label` field is required for v-select display

`LazySelect.fetchTasks` maps raw API task objects to add a computed `label` before setting `this.options`:
```javascript
{ ...task, label: `${task.task_name} (${task.project.name})` }
```
v-select renders using `label`, not `task_name`. Any code setting `lazySelect.options` directly must call `labelledTask()` first, and the API request must include `with: ['project']` for the project name to be available. (Confirmed by grepping the minified `app.js`.)

### `vSelectComp.select()` vs `lazySelect.inputHandler()`

- `inputHandler(id)` â€” updates the parent form's reactive data (task_id) but does NOT update v-select's visual state. Useful if you only need the form value.
- `vSelectComp.select(option)` â€” triggers the full event chain: v-select emits `input` â†’ LazySelect calls `inputHandler` â†’ parent form updates. Both visual state and form data update correctly. Required for post-creation auto-selection.

### `fetchTasks('')` returns nothing

The LazySelect's original `onSearch` guard (`query.length >= 3`) exists because the backend returns no results for empty or very short queries. Calling `fetchTasks('')` clears `lazySelect.options`. Initial task loading on focus must use a direct `apiFetch` call with no search filter instead.

### Placeholder check via DOM, not Vue internal

`lazySelect.value` is unreliable for detecting selected state (internal prop name varies). Checking `!!lazySelect.$el.querySelector('.vs__selected')` is stable â€” v-select always renders this element when a value is selected.

## Key Technical Notes

- **Scoped CSS can't be reused**: `team-avatars__placeholder` CSS uses Vue scoped attribute selectors (`[data-v-34a45e46]`) â€” injected DOM nodes don't carry that attribute, so styles must be inlined
- **AT-UI active state uses two classes**: `at-menu__item--active` on `<li>` AND `router-link-active` on `<a>` â€” both must be suppressed when overriding active state
- **Capture-phase listeners beat AT-UI**: AT-UI registers click handlers in bubble phase; adding `{ capture: true }` + `stopImmediatePropagation` lets our handler win
- **`localStorage.dashboard.tab` guard**: the compiled router's `beforeEnter` on `/dashboard` reads this key and redirects to `/dashboard/team` if the value is `'team'` â€” locking it to `'timeline'` on every tick defeats this without touching compiled code
- **Tasks Users column structure** (from compiled `app.js`): `div.flex.flex-gap.flex-wrap` > `AtTooltip` (one per user) > `UserAvatar` (`vue-avatar--wrapper`) â€” no built-in truncation
