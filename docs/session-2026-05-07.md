## What Was Done

### C-012 — Time Interval Form Fixes (iterative, continued from 2026-05-06)

Several iterations were needed to get the task search UX fully working. All changes are in `app/public/time-interval-helpers.js`.

#### Issue 1: "Sorry, no matching options" on focus

`onSearch` was changed to call `fetchTasks(query, loading)` unconditionally (including empty query). The backend returns nothing for empty string searches, which wiped `lazySelect.options` every time the field received focus. Fixed by reverting `onSearch` to `query.length >= 1` and adding a separate `loadInitialTasks()` function for the focusin handler.

#### Issue 2: White space in dropdown on focus

`loadInitialTasks()` was calling `POST /api/tasks/list` and setting `lazySelect.options = data.data` directly. The dropdown showed items but with blank text.

**Root cause (found by grepping compiled `/app/public/dist/app.js`):** `LazySelect.fetchTasks` maps raw API task objects to add a computed `label` field before setting options:
```javascript
{ ...task, label: `${task.task_name} (${task.project.name})` }
```
v-select renders using `label`, not `task_name`. Setting raw API objects without this field produced blank entries.

**Fix:** Added `labelledTask(task)` helper that mirrors what `fetchTasks` does. All code that sets `lazySelect.options` directly now runs through `labelledTask()`. The API request also includes `with: ['project']` so project name is available.

#### Issue 3: Placeholder "Search tasks…" visible behind selected task

`patchPlaceholder()` was setting `.vs__search` placeholder to `'Search tasks…'` every tick unconditionally. v-select keeps the search input in the DOM even after selection — our tick kept overwriting the placeholder to be visible.

**Fix:** Check `!!lazySelect.$el.querySelector('.vs__selected')` instead of `lazySelect.value` (internal prop name is unreliable). Set placeholder to `''` when a task is selected, `'Search tasks…'` when empty.

#### Issue 4: Newly created task didn't auto-populate when another task was already selected

`vSelectComp.select(option)` was replaced with `lazySelect.inputHandler(newTask.id)` in an earlier fix — `inputHandler` updates the parent form's reactive data but does NOT update v-select's visual state. The field appeared to ignore the new task.

**Fix:** Reverted to `vSelectComp.select(labelledTask(newTask))`. This triggers the full event chain: v-select emits `input` → LazySelect calls `inputHandler` → parent form updates. Both visual state and form data update correctly. The earlier bug with this approach was caused by passing a custom option object without the `label` field — now fixed by `labelledTask()`.

#### Issue 5: Can't select existing tasks / can't remove after creation

After task creation with `vSelectComp.select()`, `_dn_initial_loaded` is reset to `false`. On next focus, `loadInitialTasks()` fires and reloads the full task list (which now includes the newly created task). This restores normal dropdown behaviour.

## Key Technical Notes

### v-select `label` field (confirmed from compiled app.js)
`LazySelect.fetchTasks` maps results to add `label = task_name + ' (' + project.name + ')'` before setting `this.options`. v-select renders using `label`. Any code setting `lazySelect.options` directly must call `labelledTask()` first, and the API request must include `with: ['project']`.

### `vSelectComp.select()` vs `lazySelect.inputHandler()`
- `inputHandler(id)` — updates parent form data only. v-select visual state does not update.
- `vSelectComp.select(option)` — triggers full chain: v-select → LazySelect → parent form. Both visual and data update. Use this for programmatic selection.

### `fetchTasks('')` returns nothing
The `onSearch` guard (`query.length >= 1`) exists because the backend returns no results for empty queries. Never call `fetchTasks('')` — use a direct `apiFetch` call with no search filter for initial task loading on focus.

### Placeholder detection via DOM
`lazySelect.value` is unreliable for detecting selected state. Use `!!lazySelect.$el.querySelector('.vs__selected')` — stable across Vue re-renders.
