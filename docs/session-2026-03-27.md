### Research completed

#### Role system (confirmed from source)
| Role | role_id | Notes |
|---|---|---|
| ADMIN | 0 | Full bypass on all policy checks via `before()` |
| MANAGER | 1 | Can create time for others, cannot delete others' time |
| USER (employee) | 2 | Default role — owns only their own data |
| AUDITOR | 3 | Read-only access |

#### TimeIntervalPolicy (confirmed from source)
- `before(User $user)` — returns `true` for ADMIN (role_id=0), `null` for all others
- `update()` — returns `$user->id === $timeInterval->user_id` — employees can edit their OWN entries
- `destroy()` — same as update — employees can delete their OWN entries
- `bulkUpdate()` / `bulkDestroy()` — iterate and call update/destroy per item
- **C-001 fix**: change `update()` and `destroy()` to `return false` — admin bypass via `before()` is unaffected

#### Delete time entry — UI location
- Delete button lives ONLY inside `ScreenshotModal.vue`
- Controlled by `canRemove` prop — defaults to `true`, no role check
- Path: **Screenshots page → click a screenshot → trash icon in modal footer**
- No delete button anywhere in the time entries list view

#### Edit time entry — UI gap
- No edit form for existing time intervals exists anywhere in the admin UI
- Backend supports it — `EditTimeIntervalRequest` → `$user->can('update', ...)` → admin bypasses
- Workaround: delete the wrong entry + add new manual entry with correct times
- Tracked as C-004

#### Employee project/task creation
- Backend returns `Unauthorized` when employee tries to create a project (confirmed during evaluation)
- `ProjectPolicy.php` and `TaskPolicy.php` control this — not yet extracted/reviewed
- Tracked as C-002 — needs policy investigation before implementing

#### DigitalOcean deployment
- Droplet size: $12/month (2 GB RAM) sufficient for 10 users
- Stack on server: Docker + host Nginx (reverse proxy) + Certbot (free SSL)
- Port setup: container binds to `127.0.0.1:8080`, host Nginx proxies from 443
- `.env` always created manually on the server — never committed to git
- Git clone approach recommended over manual upload for easier updates
